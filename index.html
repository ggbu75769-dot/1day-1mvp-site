무엇을 확인하나요?

테스트 누락, 문서와 코드의 어긋남, 공개되면 안 되는 값, 출시 파일 범위, 승인 필요 항목을 함께 봅니다. 차단 항목이 있으면 고친 뒤 다시 확인합니다.

  • 문서와 실제 동작의 어긋남
  • 테스트 또는 출시 파일의 빈틈
  • 공개 사이트에 들어가면 안 되는 값
  • 승인 전 멈춰야 하는 결정

안전하게 나눈 공개 범위

공개 사이트는 소개만, 실제 실행은 비공개로 둡니다.

GitHub Pages에는 정적 소개 파일만 올립니다. 실제 실행, 승인 처리, API 호출, 운영 기록은 공개 사이트에 두지 않습니다.

자동화가 혼자 넘기지 않는 결정

결제, 개인정보, 실제 배포, 유료 외부 API, 도메인과 DNS, 저장소 공개, 라이선스 선택, 취약점 신고 채널은 승인 전까지 멈춰 둡니다.

  • 정적 공개 사이트만 운영
  • Runtime API는 공개 배포하지 않음
  • 관리 콘솔은 비공개 환경에서 확인
  • 소스 공개와 라이선스는 승인 대기

FAQ

방문자가 먼저 알아야 할 것

서비스 소개는 선명하게, 공개 범위는 정확하게 보여줍니다.

지금 바로 가입할 수 있나요?

아직은 받을 수 없습니다. 파일럿이나 공개 데모가 준비되면 이 페이지에서 다음 단계를 안내하겠습니다.

이 페이지에서 데이터를 저장하나요?

아닙니다. 이 페이지는 폼 제출이나 백엔드 연결이 없는 정적 GitHub Pages 소개 사이트입니다.

누구에게 맞나요?

1인 창업자, 인디 메이커, 소규모 제품팀처럼 AI와 함께 작은 실험을 만들고 결과까지 남기고 싶은 팀에 맞습니다.

모든 MVP가 하루 안에 출시되나요?

아닙니다. 목표는 하루 안에 확인 가능한 작은 범위로 줄이고, 만들고, 결과를 보고, 다음 결정을 남기는 것입니다.

비공개 미리보기

다음 공개 단계가 열리면 이 페이지에서 먼저 안내합니다.

이 페이지는 공개 소개 사이트입니다. 실제 관리 콘솔은 비공개 파일럿 환경에서만 확인합니다.
