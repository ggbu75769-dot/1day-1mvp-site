Workflow Spine

운영 로직

1Day-1MVP는 아이디어를 바로 개발로 던지지 않고, 범위 잠금과 검증 게이트를 통과한 작업 패킷으로 바꿉니다.

아이디어 수집

실행 가능한 MVP 후보를 모으고 하루 안에 검증할 수 있는 문제로 좁힙니다.

범위 잠금

오늘 만들 기능, 보류할 기능, 성공 판단 기준을 한 번에 고정합니다.

Codex 작업 패킷 생성

구현자가 바로 실행할 수 있도록 목적, 파일, 게이트, 검증 명령을 묶습니다.

구현 및 변경 추적

작업 결과와 실행 영수증을 프로젝트 흐름에 연결해 변경 맥락을 잃지 않습니다.

QA-Z / 릴리스 검증

보안, 문서, 테스트, 릴리스 안정성 검사를 통해 병합 가능성을 판단합니다.

인간 승인 게이트

결제, 개인정보, 외부 API, 도메인, 공개 릴리스는 자동화 밖에서 승인합니다.

출시 증거 기록

공개 URL, 결과, 차단 사유, 검증 결과를 다음 판단이 가능한 형태로 남깁니다.

결과 리뷰와 다음 결정

출시 후 데이터를 검토하고 반복, 보류, 템플릿화 중 다음 결정을 선택합니다.

Human Gates

승인 게이트

결제, 개인정보, 외부 API, 도메인/DNS, 공개 소스 릴리스 같은 위험한 행동은 자동화가 아니라 인간 승인 뒤에 둡니다.

안전 기능입니다

  • source-release approval is pending
  • LICENSE is absent because license metadata is not approved
  • runtime-api is not deployed publicly
  • This is a safety feature, not a bug

Operator Notes

FAQ / 운영자 노트

공개 사이트가 무엇을 제공하고 무엇을 의도적으로 제공하지 않는지 명확히 정리합니다.

이 사이트에서 실제 앱을 사용할 수 있나요?

이 사이트는 제품 로직과 출시 상태를 설명하는 정적 공개 페이지입니다. 실제 프로젝트 관리 대시보드는 공개 Pages가 아니라 비공개 console-web 앱의 /manage에서 별도로 제공합니다.

GitHub Pages에 백엔드도 올라가 있나요?

아니요. GitHub Pages는 정적 파일만 호스팅하며 runtime-api는 배포되어 있지 않습니다.

왜 소스 공개 릴리스는 대기 중인가요?

저장소 공개, 라이선스 선택, 취약점 제보 채널은 인간 승인 전까지 진행하지 않기 때문입니다.

QA-Z는 무엇을 검증하나요?

보안, 릴리스 안정성, 문서 정합성, 테스트 품질, 병합 가능성을 공개 가능한 수준으로 설명합니다.

1Day-1MVP는 누구를 위한 도구인가요?

하루 단위로 MVP를 고르고, 범위를 잠그고, Codex 작업을 발행하고, 증거 기반으로 다음 결정을 내려야 하는 운영자를 위한 도구입니다.
